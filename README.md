# 🌤️ Метеостанція на основі ESP32

> IoT-система автоматизованого моніторингу метеорологічних параметрів із хмарним зберіганням даних, MQTT-інтеграцією та локальним веб-інтерфейсом реального часу.

## 📌 Загальна інформація

- **Тип проєкту**: Вбудована IoT-система (Embedded / IoT)
- **Мова програмування**: C++ (Arduino IDE), HTML5 / CSS3 / JavaScript
- **Мікроконтролер**: ESP32-WROOM-32 (Espressif Systems)
- **Датчики**: BME280 (температура, вологість, тиск), MH-RD (опади)
- **Хмарні сервіси**: Firebase Realtime Database, HiveMQ (MQTT-брокер)

## 🧠 Опис функціоналу

- 🌡️ Вимірювання температури (±1 °C), вологості (±3 %), атмосферного тиску та рівня опадів у реальному часі
- 📡 Передавання телеметрії в хмару через **Firebase RTDB** (кожні 5 секунд) та **MQTT** (брокер HiveMQ)
- 🌐 Вбудований **HTTP веб-сервер** (SPA) з доступом із будь-якого браузера в локальній мережі
- 📊 Інтерактивні графіки на **HTML5 Canvas** з тултіпами та 24-годинною історією
- 📈 Сторінка статистики: мінімум, середнє, максимум для кожного параметра
- 🌙 Підтримка **темної та світлої теми** інтерфейсу
- ⏰ Синхронізація реального часу через **NTP** (часовий пояс: Україна, EET/EEST)
- 💡 Індикація дощу через **вбудований LED** (GPIO 2)
- 📶 Налаштування Wi-Fi без перепрошивки — через портал **WiFiManager**

## 🧱 Опис основних файлів / модулів

| Файл / Модуль | Призначення |
|---|---|
| `ESP_Weather_station.ino` | Головний файл прошивки ESP32 |
| `index_html[]` (PROGMEM) | Вбудований HTML/CSS/JS веб-інтерфейс (SPA) |
| `handleRoot()` | Обробник HTTP GET `/` — повертає веб-сторінку |
| `handleData()` | Обробник HTTP GET `/data` — повертає JSON з поточними даними та статистикою |
| `handleHistory()` | Обробник HTTP GET `/history` — повертає JSON з 24-годинною історією |
| `getTimeString()` | Функція отримання поточного часу через NTP |
| `getRain()` | Зчитування та інвертування значення АЦП датчика опадів (GPIO 34) |
| `setup()` | Ініціалізація: Wi-Fi, BME280, Firebase, MQTT, NTP, веб-сервер |
| `loop()` | Основний цикл: оновлення даних кожні 5 с, запис в історію кожні 60 с |

## ▶️ Як розгорнути проєкт "з нуля"

### 1. Необхідне програмне забезпечення

- **Arduino IDE** v2.x або пізніше — [завантажити](https://www.arduino.cc/en/software)
- Підтримка плат **ESP32** від Espressif: додати в менеджер плат URL `https://dl.espressif.com/dl/package_esp32_index.json`

### 2. Встановлення бібліотек

Через **Менеджер бібліотек** (Sketch → Include Library → Manage Libraries) встановити:

- `Adafruit BME280 Library` (+ залежність `Adafruit Unified Sensor`)
- `PubSubClient` (Nick O'Leary)
- `WiFiManager` (tzapu)
- `Firebase ESP Client` (Mobizt)

### 3. Налаштування Firebase

1. Зайти на [console.firebase.google.com](https://console.firebase.google.com) і створити новий проєкт.
2. Увімкнути **Realtime Database** (режим: тестовий або з правилами).
3. Скопіювати **Database Secret** (розділ "Налаштування проєкту → Сервісні акаунти → Database secrets").
4. У файлі `meteostation.ino` замінити значення констант:

```cpp
#define API_KEY      "ВАШ_DATABASE_SECRET"
#define DATABASE_URL "ВАШ_ПРОЄКТ-default-rtdb.europe-west1.firebasedatabase.app"
```

### 4. Завантаження прошивки

1. Підключити плату ESP32 до ПК через USB.
2. У Arduino IDE обрати плату: `Tools → Board → ESP32 Dev Module`.
3. Обрати правильний COM-порт: `Tools → Port`.
4. Натиснути `Upload` (Ctrl+U).

### 5. Перше підключення до Wi-Fi

Після першого завантаження ESP32 створить точку доступу **`Meteo_Maksym_Setup`**:

1. Підключитися до цієї мережі з телефону або ноутбука.
2. Відкрити браузер і перейти за адресою `192.168.4.1`.
3. Обрати свою Wi-Fi мережу, ввести пароль і натиснути "Save".
4. ESP32 перезавантажиться і підключиться до вашої мережі.

> **Примітка:** Після успішного підключення IP-адресу пристрою буде виведено в Serial Monitor (швидкість: 115200 baud).

### 6. Відкриття веб-інтерфейсу

Відкрити браузер і перейти за IP-адресою ESP32, наприклад: `http://192.168.1.100`

## 🌐 REST API (локальний веб-сервер)

### GET `/data` — поточні дані та статистика

**Відповідь (JSON):**

```json
{
  "t": 23.4,    "h": 58.2,    "p": 752.1,   "r": 45,
  "mt": 18.1,   "xt": 25.7,   "at": 22.3,
  "mh": 45.0,   "xh": 72.5,   "ah": 59.1,
  "mp": 748.3,  "xp": 755.6,  "ap": 751.8,
  "mr": 0,      "xr": 312,    "ar": 28.5
}
```

| Ключ | Опис |
|---|---|
| `t`, `h`, `p`, `r` | Поточні: температура (°C), вологість (%), тиск (mmHg), опади (АЦП) |
| `mt`/`xt`/`at` | Мін/макс/середня температура |
| `mh`/`xh`/`ah` | Мін/макс/середня вологість |
| `mp`/`xp`/`ap` | Мін/макс/середній тиск |
| `mr`/`xr`/`ar` | Мін/макс/середній рівень опадів |

### GET `/history` — 24-годинна історія (запис щохвилини)

**Відповідь (JSON):**

```json
{
  "t": [20.1, 20.5, ..., 23.4],
  "h": [60.0, 61.2, ..., 58.2],
  "p": [750.1, 750.3, ..., 752.1],
  "r": [0, 0, ..., 45]
}
```

Кожен масив містить до 24 останніх значень (або `-999` для ще не заповнених слотів).

## 🖱️ Інструкція користувача (веб-інтерфейс)

Інтерфейс складається з трьох вкладок:

**1. "Зараз"** — відображає актуальні показники:
- Значення температури, вологості, тиску та опадів оновлюються кожні **5 секунд**
- Під кожним показником — інтерактивний графік (остання 15 точок)
- Кольоровий бейдж відображає стан: ХОЛОДНО / КОМФОРТНО / СПЕКА, СУХО / ВОЛОГО, ДОЩ тощо
- Наведення на точку графіка показує **тултіп** з часом та описом стану

**2. "Історія"** — графіки змін за останні 24 години (оновлення кожні 15 секунд)

**3. "Статистика"** — мінімум, середнє та максимум для всіх параметрів із початку роботи пристрою

**🌙 Перемикач теми** (кнопка у верхньому правому куті) — перемикає між темним та світлим оформленням.

## 🧪 Відомі проблеми та рішення

| Проблема | Причина | Рішення |
|---|---|---|
| ESP32 не з'являється у COM-портах | Відсутній драйвер USB-UART | Встановити драйвер CP2102 або CH340 |
| Не підключається до Wi-Fi | Невірний пароль або збережені налаштування | У коді тимчасово увімкнути `wm.resetSettings()` |
| BME280 не знайдено | Неправильна I2C-адреса | Перевірити: адреса 0x76 (SDO=GND) або 0x77 (SDO=3.3V) |
| Firebase повертає помилку | Недійсний API_KEY або DATABASE_URL | Перевірити правильність констант та правила RTDB |
| Час відображається як "Time_Syncing" | NTP ще не синхронізовано | Зачекати кілька секунд після підключення до Wi-Fi |
| Графіки не відображаються | Замало точок (менше 2) | Зачекати 2–3 цикли оновлення (10–15 секунд) |
